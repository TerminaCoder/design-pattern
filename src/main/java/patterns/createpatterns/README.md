### 创建类模式有5种，分别是：
```text
        单例模式：用于得到内存中的唯一对象。
        工厂方法模式：用于创建复杂对象。
        抽象工厂模式：用于创建一组相关或相互依赖的复杂对象。
        建造者模式：用于创建模块化的更加复杂的对象。
        原型模式：用于得到一个对象的拷贝。
```

### 使用场景：
```text
        创建类模式主要关注对象的创建，将对象的创建过程进行封装，使客户端可以直接得到对象，而不用关心如何去创建对象。
```
    
### 优点：
```text
        所有的创建类模式的本质上都是对对象的创建过程进行封装
```
